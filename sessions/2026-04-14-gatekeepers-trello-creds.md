### gh CLI multi-account quirk

10. The dev box has two gh accounts: `fos-duoc` (secondary, READ-only on `fuad-onate-evs/*`) and `fuad-onate-evs` (primary, ADMIN on the POC repo). `gh auth switch -u fuad-onate-evs` updates `gh auth status` but leaves `~/.config/gh/hosts.yml` top-level `user:` / `oauth_token:` pointing at `fos-duoc`, so both the credential helper and gh API operations use the wrong account. **Workaround** (captured in auto-memory): manually edit `hosts.yml` top-level `user:` + `oauth_token:` to the desired account. Verified after patch: `git push` and `gh pr merge` both succeed as `fuad-onate-evs`.

### Trello credentials via Power-Up

11. **User was a guest** (not admin) in the Evalueserve Workspace that owns `poc-evalueserve` — so could not create a Power-Up there. Created a personal Workspace `fuad-dev` at <https://trello.com/create-workspace>, then a Power-Up `poc-data-eng-dev` in that workspace (<https://trello.com/power-ups/admin>).
12. **API key** generated on the Power-Up's "Clave de API" tab (32 hex chars). **Secreto** on the same page is NOT the user token; it's the OAuth 1 shared secret. The **user token** (~76 chars starting with `ATTA`) is generated by the authorize flow: `https://trello.com/1/authorize?expiration=never&scope=read,write,account&response_type=token&key=<API_KEY>&name=<name>` → "Allow" → token.
